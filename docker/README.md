# Разработка с Docker

## Поднять контейнеры

```bash
docker compose up -d
```

Приложение: http://localhost:8080

При первом запуске entrypoint:
- ждёт готовности MySQL;
- при отсутствии `vendor` выполняет `composer install`;
- запускает миграции;
- стартует Apache.

При каждом перезапуске контейнера (`docker compose restart app` или `up -d`) миграции выполняются снова (уже применённые не дублируются).

## Разработка с локалки

В `docker-compose.yml` примонтирован том:

```yaml
volumes:
  - .:/var/www/html
```

Текущая папка проекта на хосте — это и есть `/var/www/html` внутри контейнера. Поэтому:

- правишь файлы в IDE на своём компьютере;
- сохраняешь — изменения сразу видны в контейнере;
- обновляешь страницу в браузере — видишь результат.

Отдельно ничего не нужно «прокидывать»: код один и тот же на хосте и в контейнере.

## Полезные команды

```bash
# Логи app (в т.ч. вывод миграций)
docker compose logs -f app

# Зайти в контейнер
docker compose exec app bash

# Внутри контейнера: миграции вручную
php yii migrate

# Остановить
docker compose down

# Остановить и удалить данные MySQL
docker compose down -v
```

## База без Docker (локалка)

Если поднимаешь только MySQL в Docker, а PHP запускаешь локально (например, `php -S localhost:8080 -t web`), задай переменные окружения или отредактируй `config/db.php`:

- хост: `127.0.0.1` или `localhost`
- порт: `3306`
- база/пользователь/пароль — как в `docker-compose.yml` (ubtdatabase / ubt / secret).
