# UBTdatabase

Веб-приложение на [Yii 2](https://www.yiiframework.com/): админ-панель с ролями и управление аккаунтами Яндекс.Дзен и постами.

## Возможности

- **Авторизация** — вход по логину/паролю, сессии.
- **Роли (RBAC)** — роли `admin`, `manager`, `user`; в админке можно назначать роли пользователям.
- **Аккаунты Дзен** — CRUD аккаунтов: название, описание, ссылка на канал, тематика.
- **Посты** — посты к аккаунтам со статусами: черновик, ожидает постинга, запощено.

## Требования

- PHP 8.4+
- MySQL 8.0
- Composer

## Установка и запуск (Docker)

```bash
git clone https://github.com/skrabik/UBTdatabase.git
cd UBTdatabase

docker compose up -d
```

Приложение: **http://localhost:19080**

При первом запуске автоматически:
- поднимается MySQL;
- при отсутствии `vendor` выполняется `composer install`;
- применяются миграции (таблицы user, RBAC, zen_account, zen_post);
- создаётся пользователь **admin** / **admin** с ролью admin;
- запускается встроенный PHP-сервер (`php yii serve`).

При перезапуске контейнеров миграции выполняются снова (уже применённые не дублируются).

## Разработка с хоста

В контейнер примонтирован весь проект (`.:/var/www/html`). Редактируешь файлы локально — изменения сразу видны в контейнере, достаточно обновить страницу в браузере.

```bash
# Логи приложения
docker compose logs -f app

# Войти в контейнер
docker compose exec app bash

# Миграции вручную (в контейнере)
php yii migrate
```

## Установка без Docker

1. Установи PHP 8.4+, MySQL, Composer.
2. Склонируй репозиторий и установи зависимости:

   ```bash
   composer install
   ```

3. Настрой `config/db.php` (хост, база, пользователь, пароль MySQL).
4. Примени миграции:

   ```bash
   php yii migrate
   ```

5. Запусти сервер:

   ```bash
   php yii serve
   ```

Открой http://localhost:19080 (или 8080 при локальном `php yii serve`). Вход: **admin** / **admin**.

## Маршруты

| URL | Описание |
|-----|----------|
| `/` | Редирект: гости → вход, авторизованные → админка |
| `/admin/login` | Вход |
| `/admin` | Админ-панель (только роль admin) |
| `/admin/zen-account` | Аккаунты Яндекс.Дзен |
| `/admin/zen-post` | Посты |
| `/admin/user` | Пользователи и роли |

## Структура БД

- **user** — пользователи (логин, хеш пароля).
- **auth_*** — таблицы RBAC (роли и назначения).
- **zen_account** — аккаунты Дзен (название, описание, url, тематика).
- **zen_post** — посты (аккаунт, заголовок, текст, статус: draft / pending / posted).

Подробнее: [docs/zen-database-schema.md](docs/zen-database-schema.md).
